# comparison/ — Generative Model Benchmark

Benchmarks alternative generative models for the same conditional transition
density that the training-free diffusion flow map in `jump1d/` learns,

    p(X_{t+Delta t} | X_t)

on **exactly the same ground-truth data** — `../artifacts_bd/data_pairs.npz`,
the bounded jump-diffusion (absorbing-boundary) transition pairs produced by
`jump1d/data_generation.py`. Nothing in `jump1d/` (`common.py`,
`data_generation.py`, `training.py`, `inference.py`) is modified or
regenerated by anything in this folder.

Two models are implemented:

| Model | File | Objective |
|---|---|---|
| Conditional RealNVP (normalizing flow) | `models/realnvp.py` | exact log-likelihood |
| Conditional DDPM (diffusion) | `models/diffusion.py` | epsilon-prediction MSE, cosine schedule |

## Quick start

```bash
cd jump1d/comparison
pip install -r requirements.txt

python train_nf.py              # -> checkpoints/realnvp.pt
python train_diffusion.py       # -> checkpoints/diffusion.pt
python compare.py               # -> results/results.{csv,json}, figures/*.{png,pdf}
```

or, equivalently, via the wrapper scripts (any `config.py` flag can be passed through):

```bash
./train_nf.sh --nf_epochs 300 --batch_size 2048
./train_diffusion.sh --diff_epochs 300 --diff_n_timesteps 1000
./compare.sh --eval_n_anchors 7
```

Everything auto-detects `cuda`/`cpu` (`config.device = "auto"`) and uses
`torch.cuda.amp` mixed precision on CUDA by default (`config.amp = True`).


## Dataset

`config.data_path` defaults to `../artifacts_bd/data_pairs.npz` (keys
`x_sample` = X_t, `y_sample` = X_{t+Delta t}, both float32 scalars in the
bounded domain `[0, 6]`). To benchmark on a different dataset — e.g. a future
higher-dimensional or unbounded SDE — point `--data_path` at any `.npz` with
two equally-shaped arrays and, if the key names differ, override `--x_key` /
`--y_key`. The rest of the pipeline (models, training, metrics, figures) is
dimension-agnostic.

If a sibling `data_exit.npz` exists next to `data_path` (as it does for the
bounded model), `evaluate.py` uses it to build an empirical `P(exit | X_t)`
lookup so the multi-step rollout figure can fairly account for absorption —
none of the three rollout methods (Ground Truth NN-bootstrap, RealNVP,
DDPM) otherwise model the exit event, exactly like the flow map in `jump1d`.

## Metrics (`results/results.csv`, `results/results.json`)

Computed against the held-out validation split (paired one ML draw per
validation `X_t`, so the conditioning marginal matches exactly):

- Wasserstein-1 distance
- KL divergence (histogram-based)
- Hellinger distance
- MMD (RBF kernel, unbiased estimator)
- Sampling speed (samples/sec)
- Training time (wall-clock, recorded during `train_*.py`)
- Peak GPU memory during training
- Negative log-likelihood — **RealNVP only** (exact, via change-of-variables;
  diffusion models don't expose a closed-form NLL, so this field is blank
  for the DDPM row, as specified)

## Figures (`figures/`)

1. `01_histograms` — Ground Truth vs. RealNVP vs. DDPM histograms at several `X_t` anchors
2. `02_pdf_overlay` — KDE-smoothed density overlay at the same anchors
3. `03_qq_plot` — quantile-quantile, RealNVP vs. GT and DDPM vs. GT
4. `04_cdf_comparison` — empirical CDF overlay (aggregate)
5. `05_wasserstein_bar` — Wasserstein-to-GT, grouped by anchor + overall
6. `06_training_loss` — training/validation curves for both models
7. `07_sampling_speed` — samples/sec bar chart
8. `08_gpu_memory` — peak training GPU memory bar chart
9. `09_rollout_comparison` — composed multi-step rollout: survival fraction + terminal density

## Adding another model (GAN, VAE, Flow Matching, ...)

1. Implement it under `models/your_model.py` with a `sample(x_cond) -> y` method
   (and, optionally, a `log_prob` for exact-NLL reporting).
2. Add a `train_your_model.py` mirroring `train_nf.py` / `train_diffusion.py`:
   build the model, train with the `Config`/`dataset.py` plumbing already in
   place, and `save_checkpoint(...)` with `model_config`, `x_norm`, `y_norm`,
   `history`, `training_time_s`, `peak_gpu_mem_bytes`.
3. Add a `load_your_model` + a `sample_your_model_raw` helper to `evaluate.py`
   (copy the `nf`/`diffusion` branches in `evaluate_model`, `evaluate_anchors`,
   `evaluate_rollout`), and extend `compare.py`'s `rows`/figure calls.

No other file needs to change — `config.py`, `dataset.py`, `metrics.py`, and
`visualize.py` are all model-agnostic.
